# Joseph Gitau — Portfolio Site (LLM Context)

## Site Overview
Personal portfolio of **Joseph Gitau (mfalme0)**, a full-stack engineer based in Nairobi, Kenya.  
**URL:** https://josephgitauc.vercel.app  
**Stack:** Next.js 16 (App Router), React 19, Tailwind CSS 4, TypeScript, Framer Motion, Three.js

---

## Pages & Routes

| Route | Description | SEO Metadata |
|-------|-------------|-------------|
| `/` | Single-page horizontal-scroll portfolio: Hero → About Me → Experience → Tech Stack → Languages → Projects → Testimonials → GitHub → End | Full OG, JSON-LD (Person schema) |
| `/gear` | Gear showcase: 11 items across 6 categories (system, keyboard, mouse, audio, display, power) | Title + description + OG |
| `/gear/[slug]` | Individual gear detail page | Dynamic title/description + OG |
| `/homelab` | Homelab dashboard: server services, status indicators, resource monitors | Title + description + OG |
| `/homelab/[slug]` | Individual homelab service detail page | Dynamic title/description + OG |
| `/LAN` | LAN party photo gallery with lightbox | Title + description + OG |

All sub-routes have dedicated `layout.tsx` with proper metadata exports.

---

## Home Page Sections (in order, horizontal scroll)

1. **Hero** – Name, typewriter role animation, stats counters (3+ years, 20+ projects, full-stack delivery), GitHub/LinkedIn/Resume links
2. **About Me** – 4 expandable domain cards (Frontend, Backend, Mobile, DevOps)
3. **Experience** – Work history timeline with role details
4. **Tech Stack** – Grid of technologies with icons (3 columns sm:4 md:5)
5. **Languages** – Language proficiency bars with percentage scores
6. **Projects** – Filterable cards (All, Web, Mobile, CLI, Backend) with screenshots, tech tags, links
7. **Testimonials** – Client quote carousel with prev/next navigation
8. **GitHub** – Contribution calendar widget from react-github-calendar
9. **End** – Marquee tech ticker, contact form, social links, footer

Navigation: Fixed header with nav links + keyboard arrows + touch swipe + scroll wheel.

## SEO Implementation Details

### Structured Data (JSON-LD)
- **Person schema** with name, jobTitle, url, sameAs (GitHub, LinkedIn, X), knowsAbout skills
- Injected in root layout `<head>` via `<script type="application/ld+json">`

### Metadata (Next.js Metadata API)
- **Root layout**: Template "%s — Joseph Gitau", default description, keywords array, OpenGraph + Twitter cards, robots (index/follow)
- **Gear detail**: `generateMetadata` with item name + description
- **Homelab detail**: `generateMetadata` with service name + description
- **Gear/Homelab/LAN list**: Layout-level metadata export

### Sitemap (`/sitemap.ts`)
- Static pages: `/` (1.0), `/gear` (0.9), `/LAN` (0.7)
- Dynamic: All `/gear/[slug]` pages (0.6), all `/homelab/[slug]` pages (0.6)
- Change frequencies: monthly for static, weekly for gear/LAN, monthly for details

### robots.txt (`/robots.ts`)
- Allows all user agents
- Points to `/sitemap.xml`
- Host: https://josephgitauc.vercel.app

### OpenGraph Image
- Dynamic OG image generated via `app/opengraph-image.tsx` (Next.js `ImageResponse`)
- 1200×630 PNG with name, title, and tech stack on dark background
- Referenced across all pages: root layout, gear list, gear detail, homelab list, homelab detail, LAN
- Fallback: text-based metadata (title + description) for platforms that don't fetch images

### Technical SEO
- `metadataBase` set to canonical URL
- `alternates.canonical` set on every page (root, gear detail, homelab detail)
- `suppressHydrationWarning` on `<html>` for theme script
- Skip-to-content link for accessibility
- Semantic HTML sections with IDs for anchor navigation
- Responsive viewport (default Next.js)
- Fast loading with Geist font via next/font (no CLS)
- `googleBot` extended rules: `max-image-preview: large`, `max-snippet: -1`

## Gear Data (`/gear` and `/gear/[slug]`)

11 items with the following structure:
- **name, brand, category, slug, image, price, currency, link**
- **specs:** Array of `{ label, value, icon }` (CPU, RAM, GPU, connectivity, etc.)
- **story:** Personal narrative about the gear
- **pros / cons:** Arrays of strings
- **related:** Array of related gear slugs

Categories: `system`, `keyboard`, `mouse`, `audio`, `display`, `power`

---

## Homelab Data (`/homelab` and `/homelab/[slug]`)

Services with the following structure:
- **name, slug, description, status (operational/degraded/down/maintenance), type, icon, url, tags**
- **stats:** Array of `{ label, value, unit }` (uptime, requests, latency, etc.)
- **history:** Array of `{ date, type (info/warning/error), message }`
- **related:** Array of related service slugs

## Theming & UX
- **Dual theme:** Dark (default) + Light, toggle via header button, persisted to localStorage
- **Accent color:** `#E8A33D` (amber/gold)
- **Navigation:** Custom horizontal scroll (`translateX`), keyboard arrows, touch swipe, scroll wheel
- **Drawing overlay:** PencilCanvas persists across all pages via SketchProvider in layout, toggle with `Ctrl+``
- **Page loaders:** Themed boot sequences (home = typographic, gear = ROG Armoury Crate, homelab = BIOS POST, LAN = network scan)
- **Adaptive performance:** Detects device memory/CPU to adjust animation quality
- **Font:** Geist Sans + Geist Mono (variable fonts via next/font)
- **Responsive:** Full mobile support with hamburger nav
